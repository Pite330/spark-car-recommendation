# Project Rules

本项目是独立的消费者购车推荐系统，不默认引用其他项目的代码、数据或目录结构。

## 产品范围

- 只实现消费者输入条件、车型推荐、推荐原因、车型对比和必要可视化。
- 不增加注册、登录、用户中心、管理员后台、订单和厂商决策功能。
- 新功能先检查是否属于 `docs/MVP.md`；超出范围时先更新需求，不直接扩张实现。

## 技术约束

- Spark 必须承担真实的数据清洗、特征处理或统计任务，并保留可演示的运行结果。
- 推荐车型由确定性的筛选与评分算法产生。
- 大模型只能改写结构化推荐依据，禁止补充数据集中不存在的参数或替换推荐车型。
- 大模型调用必须有超时、异常处理和模板化降级。
- Flask 仅作为轻量接口和页面服务，不建设复杂后台。

## 数据约束

- `data/raw/` 保存原始数据，禁止在原文件上直接修改。
- `data/processed/` 只保存可由脚本重新生成的处理结果。
- 数据字段变化时同步更新 `docs/data-dictionary.md`。
- 缺失字段应取消对应规则或评分项，禁止虚构数据补齐。

## 协作约束

- 模块边界和 JSON 字段以 `docs/api-contract.md` 为准。
- 算法权重变化时同步更新 `docs/recommendation-design.md`。
- 每次提交只包含一个相对独立的模块或文档变更。
- 不提交 API Key、密码、个人令牌、原始大体积数据和本地环境文件。
