## 2. 可直接复用的项目描述

### 2.1 项目背景

汽车品牌、价格和能源类型信息分散，普通消费者难以在有限时间内从大量车型中筛选出符合预算和使用场景的候选车型。本项目面向近期有购车需求但缺少汽车专业知识的消费者，使用 Spark 对公开车型快照进行清洗和特征处理，再通过确定性的条件筛选与加权评分返回 3—5 款车型及可追溯的推荐依据。

### 2.2 项目目标

完成一条可运行、可演示、可复现的消费者购车推荐链路：用户输入预算、车型、能源、品牌、场景和座位条件，系统返回推荐车型、匹配得分和推荐原因，并支持对 2—3 款车型进行参数对比。网络或大模型不可用时，核心推荐与模板化说明仍能正常工作。

### 2.3 项目范围

项目只包含数据采集快照、Spark 离线处理、条件输入、确定性车型推荐、推荐解释、车型对比和必要可视化。不包含注册登录、用户中心、管理员后台、订单支付、经销商对接、厂商生产决策和复杂深度学习训练。

### 2.4 技术亮点

1. Spark 承担真实的清洗、枚举映射、异常校验、衍生特征和统计输出；
2. 推荐结果由确定性规则与评分算法产生，相同输入结果一致；
3. 数据字段缺失时自动取消对应评分项并重算权重，不虚构参数；
4. DeepSeek 只润色算法已生成的结构化依据，超时或失败自动降级；
5. 原始 CSV、标准 CSV/Parquet、统计 JSON 和 SHA-256 元数据形成可验证的数据处理证据链。

### 2.5 已知限制

当前数据是 2026-07-10 的有限样本快照，不是实时全量车市数据。指导价不等于成交价，月销量不等于官方上牌量；来源未稳定提供续航和油耗时系统保持空值。Flask 开发服务适合本地或局域网演示，不直接作为公网生产服务。

### 3. 数据存储设计说明书

模板包含数据存储技术选型、部署视图、结构化/非结构化/列式/分布式存储、结构设计和数据字典。

本项目没有业务数据库，必须如实说明：

- 原始层：`data/raw/16888_cars_snapshot.csv`，只读快照；
- 标准层：`data/processed/cars.csv`，供 Flask 与推荐算法读取；
- 列式层：`data/processed/cars.parquet/`，作为 Spark 处理结果；
- 证据层：`stats.json` 和 `metadata.json`；
- 无 MySQL 表、无用户账户数据、无个人信息；
- 字段定义引用 `docs/data-dictionary.md`，采集与边界引用 `docs/data-acquisition-16888.md`。

模板中 MySQL、HBase、HDFS 等未实际使用的内容应标注“不适用”，不要为了充页数虚构部署。

### 5. 测试用例

模板每条用例包含编号、版本、环境、前提条件、步骤和预期输出。至少整理以下用例：

| 编号 | 用例 | 预期 |
| --- | --- | --- |
| TC-DATA-01 | 运行 Spark 清洗 | 输入/输出 190/190，生成 CSV、Parquet、统计和元数据 |
| TC-REC-01 | 15—20 万、纯电 SUV、城市通勤、5 座 | 返回 3—5 款，字段完整，有至少两条依据 |
| TC-REC-02 | 重复相同请求 | 车型顺序和分数一致 |
| TC-REC-03 | 预算下限高于上限 | 返回 400 和字段级错误 |
| TC-REC-04 | 严格条件候选不足 | 仅按设计顺序放宽并返回 `relaxed_conditions` |
| TC-REC-05 | 来源缺少续航/油耗 | 取消对应评分项，仍能推荐 |
| TC-CMP-01 | 对比 2—3 款车型 | 返回数据集中存在的可比字段 |
| TC-CMP-02 | 对比少于 2 款或多于 3 款 | 返回参数错误 |
| TC-LLM-01 | DeepSeek 正常 | 只改写原因，不改变车型、分数和依据 |
| TC-LLM-02 | DeepSeek 超时或禁用 | 返回模板原因，主接口仍为 200 |
| TC-API-01 | 健康检查 | 数据集已加载，车型数为 190 |
| TC-UI-01 | 首页与统计图 | 表单、推荐卡片、对比和图表无溢出或遮挡 |

最终测试环境和截图以真实运行机器为准，自动化结果引用 `docs/acceptance-report.md`。

### 7. 用户使用手册

模板原本按前台、后台和实训指导划分；本项目无后台，应重构为：

1. 系统简介和运行环境；
2. 安装依赖：`make setup`；
3. 数据更新（可选）：`make data && make spark`；
4. 本地启动：`make run`，访问 `http://127.0.0.1:5000`；
5. 局域网启动：设置 `APP_HOST=0.0.0.0` 和 `APP_PORT=5050`；
6. 填写预算、车身、能源、品牌、场景和座位；
7. 阅读推荐卡片、分数、依据和条件放宽提示；
8. 选择 2—3 款车型并查看参数表/雷达图；
9. DeepSeek 配置与模板降级；
10. 常见问题：端口占用、Java 缺失、数据未生成、API Key 未配置。

正文截图至少包含首页、推荐结果、车型对比、统计图和降级提示。
