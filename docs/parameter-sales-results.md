# 配置参数与销量关联分析结果

## 1. 运行摘要

本结果由 `make analyze` 基于 2026-07-11 保存的完整配置数据生成。为避免当前配置快照与历史销量错位，分析只使用共同最新月份 `2026-06`。

| 指标 | 结果 |
| --- | ---: |
| 同月车系样本 | 425 |
| 来源参数定义 | 187 |
| Adapter 可解析参数 | 137 |
| 通过覆盖率和方差筛选的模型参数 | 98 |
| FDR < 0.05 的单参数关联 | 21 |

## 2. 模型结果

验证采用按品牌分组的五折划分，目标为 `log(月销量 + 1)`。

| 模型 | RMSE | MAE | R² |
| --- | ---: | ---: | ---: |
| 品牌、价格、级别等控制变量 Elastic Net | 1.943 | 1.546 | 0.204 |
| 控制变量 + 配置参数 Elastic Net | 1.886 | 1.522 | 0.247 |
| 控制变量 + 配置参数随机森林 | 1.882 | 1.517 | 0.252 |

加入配置参数后，Elastic Net 的平均 R² 增加 0.044，随机森林相对基线增加 0.048。配置参数确实提供了增量信息，但主要销量差异仍不能由当前配置表解释。

## 3. 当前可见信号

随机森林中较靠前的参数包括多功能方向盘、最大扭矩、最大功率、方向盘调节、蓝牙/车载电话、整备质量、主/副驾驶座电动调节、最高车速、最大马力和轴距。

多项舒适或高阶配置在单参数 Spearman 分析中与销量呈负相关。这不应解释为“取消配置能够提高销量”。更合理的解释是：配置与价格带、车型定位、豪华程度、品牌和细分市场高度绑定，单参数相关仍包含明显混杂。

Elastic Net 在按品牌分组的第二折中 R² 接近 0，说明结果对被留出的品牌集合仍然敏感。因此第一版结果适合用于发现候选参数和设计下一轮分层分析，不适合直接形成厂商配置决策。

## 4. 面向产品的结果口径

前后端统一读取 `data/processed/analysis/consolidated_parameter_results.csv`，不再自行拼接单参数相关和模型重要度。该表覆盖全部 187 个来源参数；未入模参数仍保留，并明确给出排除原因或 `not_evaluated`。

每个参数可以展示三类当前证据：

- `association_direction`：`positive`、`negative`、`mixed`、`neutral` 或 `not_evaluated`；
- `evidence_strength`：`strong`、`moderate`、`weak` 或 `not_evaluated`；
- 原始统计量：Spearman rho、FDR q 值、标准差缩放后的 Elastic Net 系数、随机森林结点不纯度重要度。

其中随机森林重要度没有方向，并可能偏向连续变量，不能单独渲染为“销量影响率”。Elastic Net 系数来自全样本拟合，适合辅助判断方向，不是交叉验证后的稳定效应。Spearman/FDR 是未控制混杂的单参数结果。产品界面应同时展示这些限制。

当前所有参数的 `trend_status` 均为 `insufficient_history`。本数据只有 2026 年 7 月保存的一次配置快照，不能据此生成“未来增长配置”“衰退配置”等结论；历史销量本身不能补足历史配置状态。未来积累可比快照后，可先输出 `proxy_only` 的渗透变化信号，再评估是否升级为趋势结论。
