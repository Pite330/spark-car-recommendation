# 配置参数与销量关联分析设计

## 1. 分析目标

分析车主之家公开配置参数与车系月销量之间的统计关联，判断配置参数在控制品牌、价格、级别、能源、车身、年款和配置款数量后，能否增加对销量差异的解释能力。

本分析不将相关性表述为因果影响。促销、渠道、供给、上市时间、地区和宏观市场等变量未被完整观测，因此结果不能直接用于断言“增加某项配置会导致销量提高”。

## 2. 时间与粒度

配置数据是抓取时点的在售配置款快照，销量是车系月度数据。第一版只使用全表共同最新月份 `2026-06`，避免用当前配置解释更早年份的销量。

配置款参数按车系聚合：

- 数值参数取车系内配置款中位数；
- 标配记为 1、选配记为 0.5、组合状态记为 0.75、明确未配备记为 0；
- 装备参数取同车系全部配置款的平均覆盖分；
- 空字符串仍视为缺失，不能与网页中的 `-`（未配备）混为一类。

## 3. 参数 Adapter

Adapter 根据实际值分布将 187 个参数分为数值型、装备型和枚举型。第一版模型使用数值型和装备型参数；高基数枚举字段保留在质量表中，待建立可靠的归一化字典后再进入模型。

以下字段禁止作为配置特征：

- `全国4S最低报价`：可能同时受到销量和市场热度影响；
- `车款人气`：与分析目标存在明显信息泄漏；
- `厂商指导价(元)`：只作为控制变量，避免在完整模型中重复进入。

进入模型的参数必须满足：在同月样本中覆盖率不低于 20%、车系间存在有效方差，并通过固定上限控制特征数量。

## 4. 模型与验证

目标变量为 `log(2026-06 月销量 + 1)`。模型使用按品牌分组的五折验证，确保同一品牌不会同时出现在训练折和验证折中。

对比三组模型：

1. 控制变量 Elastic Net；
2. 控制变量加配置参数 Elastic Net；
3. 控制变量加配置参数随机森林。

Elastic Net 用于观察正则化后的方向。进入 Elastic Net 前，控制变量和参数特征都按标准差缩放，因此系数不再直接受“元、毫米、覆盖率”等原始量纲支配；但特征高度共线时，系数仍可能被压缩到 0 或在相邻参数之间重新分配，不能单独当作影响大小。

随机森林用于检查非线性关系，输出的是全样本拟合后的结点不纯度重要度。该指标无正负方向，并天然偏向可切分点较多的连续特征，因此只作为补充排序，不进入证据强度判定。参数重要度是模型中的预测贡献，不等于因果效应。

单参数分析使用 Spearman 相关，对并列值取平均秩，并对全部入模参数的检验执行 Benjamini-Hochberg FDR 校正。当前最小覆盖率为 20%，样本量足以使用大样本近似计算 p 值；极端离散装备参数仍应结合覆盖率和效应大小解释，不能只看显著性。

## 5. 综合证据标签

`consolidated_parameter_results.csv` 为前后端提供每个来源参数一行的统一结果。未进入 Adapter、被判定为泄漏或未通过覆盖率筛选的参数也会保留，并标记为 `not_evaluated`。

关联方向综合 Spearman 和 Elastic Net：

- `positive` / `negative`：单参数方向明确，Elastic Net 未给出相反方向；
- `mixed`：Spearman 与 Elastic Net 方向相反；
- `neutral`：Spearman 接近 0，且 Elastic Net 系数被压缩为 0；
- `not_evaluated`：该参数未进入当前分析。

证据强度采用保守规则：

- `strong`：FDR ≤ 0.01、`|rho| ≥ 0.2`，且 Elastic Net 非零并同向；
- `moderate`：FDR ≤ 0.05、`|rho| ≥ 0.1`，且 Elastic Net 不反向；
- `weak`：已分析但不满足上述条件；
- `not_evaluated`：没有可用统计结果。

证据等级只代表当前横截面中多种信号的一致程度，不代表因果置信度。

当前只有单期配置快照，全部参数的 `trend_status` 固定为 `insufficient_history`。年款字段不是配置生效时间，不能替代多期快照。只有积累至少多个可比时间点后，才能另行输出渗透率变化等 `proxy_only` 信号；在此之前不生成“未来上升/下降”标签。

## 6. 运行与输出

```bash
make analyze
```

输出目录为 `data/processed/analysis/`：

- `parameter_quality.csv`：参数类型、覆盖率、方差、排除原因；
- `series_features.parquet`：车系级参数特征长表；
- `analysis_dataset.parquet`：同月销量、控制变量和模型特征宽表；
- `univariate_associations.csv`：单参数 Spearman 相关与 FDR；
- `parameter_importance.csv`：Elastic Net 系数和随机森林重要度；
- `consolidated_parameter_results.csv`：供 API 和前端加载的全参数综合证据表；
- `model_metrics.json`：按品牌分组交叉验证结果和分析元数据。

综合结果表的核心字段为：

```text
feature_name, group_name, parameter_name, parameter_type,
spearman_rho, fdr_q_value, elastic_net_coefficient,
random_forest_importance, evidence_strength,
association_direction, trend_status
```

## 7. 第一版限制

- 当前只分析公开销量覆盖的同月车系，不能代表全部 1620 个车系；
- 当前配置快照没有参数生效和停售时间，不能建立可靠历史面板；
- 枚举型复合文本尚未全部结构化；
- 参数高度共线，单个参数的方向可能代表车型定位或价格带，而非该配置本身；
- 后续如积累多期配置快照，应升级为车系固定效应或配置变更事件分析。
