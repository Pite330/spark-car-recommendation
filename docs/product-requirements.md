# 产品需求说明

## 1. 项目目标

为缺少汽车专业知识的普通消费者提供简单、可解释的车型筛选与推荐服务。用户不需要注册账号，只需填写预算和少量购车偏好，即可获得 3—5 款候选车型及推荐依据。

## 2. 目标用户

- 有明确购车预算，但不了解具体车型的消费者；
- 已确定车型或能源类型，希望快速缩小选择范围的消费者；
- 希望比较少量候选车型关键参数的消费者。

## 3. 核心使用场景

用户计划购买一辆 15—20 万元的纯电 SUV，主要用于城市通勤，希望获得符合预算、车型和能源偏好的候选车辆，并了解每款车为什么被推荐。

## 4. 功能需求

### FR-01 条件输入

系统必须支持预算区间输入，并支持车型、能源类型、品牌、使用场景等可选条件。未填写条件不参与限制。

### FR-02 候选筛选

系统必须按照硬性条件排除明显不符合需求的车辆。结果不足时，只能放宽非核心条件，并向用户说明放宽内容。

### FR-03 综合排序

系统必须对候选车型执行确定性的加权评分，返回得分最高的 3—5 款车型。相同数据和输入应得到一致结果。

### FR-04 推荐展示

每款推荐车型至少展示车型名称、品牌、参考价格、车身类型、能源类型、匹配得分和两条推荐依据。

### FR-05 车型对比

用户可以选择 2—3 款推荐车型，对比价格、车型、能源类型、座位数、续航或油耗等已有数据。

### FR-06 推荐说明

系统默认通过模板生成推荐原因。可以接入大模型润色说明，但大模型不可用时不得影响推荐结果。

### FR-07 数据可视化

系统至少提供一项与消费者选择相关的图表，例如价格区间分布、能源类型分布或推荐车型参数对比。

### FR-08 厂商分析数据扩展

项目提供一条与消费者推荐快照隔离的数据采集管道，按车系保存公开参数接口的原始响应，并将基本参数、车身、发动机、变速箱、底盘、安全、操控、外部、内部、座椅、多媒体、灯光、玻璃/后视镜、空调和高科技配置等全部字段整理为“配置款 × 参数”长表。同时保存车系月销量历史，供后续参数与销量关联分析使用。

该扩展只负责可追溯数据集和探索性关联分析，不将统计相关性表述为参数对销量的因果作用，也不进入消费者推荐接口的实时请求链路。

## 5. 非功能需求

- 单次推荐在本地演示环境中应在 3 秒内返回；大模型调用时间不计入核心算法时间；
- 输入非法预算或无可用候选时，页面必须给出明确提示；
- 推荐原因必须能够追溯到输入条件和数据字段；
- 关闭网络或大模型接口后，主流程仍可演示；
- 项目不得在代码中保存密钥。

## 6. 范围外功能

- 用户账号、登录和权限；
- 后台数据管理系统；
- 厂商销量预测、生产规划和自动营销决策；
- 将配置参数与销量的统计相关性解释为因果影响；
- 支付、订单、试驾和经销商对接；
- 评论、社区和社交功能；
- 复杂深度学习训练。
