# 四人分工与开发计划

## 1. 职位与责任

| 职位 | 核心责任 | 主要交付物 |
| --- | --- | --- |
| 项目负责人兼产品与数据协调 | 控制范围；确定数据字段；维护需求和接口；组织集成、报告与答辩 | 需求文档、数据来源记录、集成版本、项目报告、答辩材料 |
| Spark 数据工程师 | 编写 PySpark 清洗和特征任务；输出标准数据与统计结果 | Spark 脚本、标准数据集、处理日志、运行截图 |
| 推荐算法与大模型工程师 | 实现筛选、评分、放宽、模板原因和可选大模型适配 | 推荐模块、单元测试、权重说明、大模型降级 |
| 前端可视化与测试工程师 | 实现条件输入、推荐卡片、车型对比和 ECharts；维护接口与验收测试 | Web 页面、图表、接口联调、测试记录、演示页面 |

## 2. 模块负责人

| 路径或文档 | 主要负责人 | 评审人 |
| --- | --- | --- |
| `data/`、`src/spark_jobs/`、`data-dictionary.md` | Spark 数据工程师 | 项目负责人 |
| `src/recommender/`、`recommendation-design.md` | 推荐算法工程师 | Spark 数据工程师 |
| `src/web/`、`api-contract.md` | 前端可视化工程师 | 推荐算法工程师 |

| `tests/` 和最终验收记录 | 前端测试工程师 | 全员 |

## 3. 开发阶段

### 阶段一：数据与接口冻结

- 选择真实数据集；
- 完成字段映射和数据质量检查；
- 使用 10—20 条样例数据冻结推荐请求与响应格式。

完成标志：四个模块使用相同字段名和示例数据。

### 阶段二：并行开发

- Spark 工程师完成标准数据输出；
- 推荐工程师基于样例数据完成纯 Python 算法；
- 前端工程师基于固定 JSON 完成页面；
- 项目负责人同步报告、数据来源和演示脚本。

完成标志：每个模块能够独立演示。

### 阶段三：联调

- 用 Spark 输出替换样例数据；
- 用真实推荐接口替换固定 JSON；
- 验证无候选、非法预算、可选字段缺失等边界；
- 完成车型对比和至少一张 ECharts 图表。

完成标志：关闭大模型后能完整演示主链路。

### 阶段四：增强与答辩

- 主链路稳定后接入大模型；
- 完成超时和模板降级；
- 保存 Spark 运行证据；
- 按 `MVP.md` 逐项验收；
- 演练答辩和故障降级流程。

## 4. 协作接口

- Spark 工程师交付标准数据文件和字段说明；
- 推荐工程师交付一个接收字典、返回结构化推荐列表的纯函数接口；
- 前端工程师只依赖 `api-contract.md`，不读取算法内部变量；
- 项目负责人负责合并字段和范围变更，不允许口头修改后不更新文档。

## 5. 最小每日同步内容

每人只汇报四项：昨天完成、今天计划、当前阻塞、是否影响接口或字段。涉及接口和字段的变更当天写入对应文档。
